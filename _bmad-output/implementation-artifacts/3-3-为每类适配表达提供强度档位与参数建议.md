# Story 3.3: 为每类适配表达提供强度档位与参数建议

Status: ready-for-dev

## Story

As a 用户,
I want 为每类适配表达查看 lite、standard、strong 三档结果和参数建议,
so that 我可以按目标模型和生成意图选择合适的输出强度。

## Acceptance Criteria

1. 每类适配表达都提供 lite、standard、strong 三档版本。
2. 每组结果至少附带 `aspect_ratio`、`seed_strategy` 和不少于 1 条使用说明。
3. 前端切换强度档位的关键交互响应满足 `<= 200ms`，且不破坏任务上下文。

## Tasks / Subtasks

- [ ] 扩展 `prompt_outputs` schema，纳入 strength variants 与 parameter recommendations。
- [ ] 为三类 adapter 生成强度档位映射规则。
- [ ] 设计参数建议视图模型，统一暴露 `aspect_ratio`、`seed_strategy`、usage notes。
- [ ] 在结果工作区实现快速切换与性能优化。
- [ ] 为变体生成和前端切换性能补齐测试。

## Dev Notes

### Technical Requirements

- 强度档位属于同一任务结果对象的子维度，不是新的任务或新的资源。
- 参数建议必须结构化输出，不能只存在自然语言说明里。
- 切换行为只改变局部展示状态，不应触发任务状态回写。

### Architecture Compliance

- 强度档位建议落在 `prompt_outputs.adapters[*].variants` 或等价稳定结构。
- 前端局部切换状态可放在 workspace store，但不能替代服务端结果。
- 保持和 Epic 3 其余 stories 同一结果视图模型。

### Library / Framework Requirements

- React Query 读取结果，Zustand/本地状态保存当前 variant。
- React 19 视图切换可优先保持轻量更新，避免不必要整页刷新。

### File Structure Requirements

- Variants logic: `src/lib/analysis-ir/adapters/variant-builder.ts`
- UI: `src/features/analysis/components/PromptStrengthSwitcher.tsx`
- Shared types: `src/lib/analysis-ir/types.ts`

### Testing Requirements

- 单测覆盖三档结果生成与参数建议结构。
- 组件测试覆盖切换响应、上下文保持、空数据回退。
- 如有性能测试基线，加入 200ms 以内交互断言。
