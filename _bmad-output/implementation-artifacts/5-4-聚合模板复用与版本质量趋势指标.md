# Story 5.4: 聚合模板复用与版本质量趋势指标

Status: ready-for-dev

## Story

As a 产品负责人,
I want 查看模板复用率、编辑率、采纳率和版本质量趋势,
so that 我可以判断当前链路是否真正提升了用户价值和风格迁移质量。

## Acceptance Criteria

1. 系统支持最近 7 天与最近 30 天窗口查看模板复用率、二次编辑率和模板采纳率。
2. 趋势视图至少展示采纳率、满意率、迁移成功率和端到端时延，并支持按日聚合和按版本窗口比较。
3. 分析面板只消费聚合数据，不直接读取任务明细 JSONB 或阶段快照原始对象。

## Tasks / Subtasks

- [ ] 定义 analytics 聚合 schema 和时间窗口参数。
- [ ] 从模板事件、任务结果和版本字段生成聚合查询。
- [ ] 更新 analytics UI，支持时间窗口和版本对比。
- [ ] 保证面板只读取聚合结果，不泄露明细 payload。
- [ ] 为聚合查询和图表渲染补齐测试。

## Dev Notes

### Technical Requirements

- 指标口径必须严格遵守 PRD，不能展示当前无法稳定计算的假指标。
- 聚合层不应直出任务明细 JSONB。
- 指标按日聚合，版本比较依赖 `schema_version` / `prompt_version`。

### Architecture Compliance

- `features/analytics/*` 只消费聚合数据。
