# Story 4.2: 检测语义冲突与可复现性问题并生成修复建议

Status: ready-for-dev

## Story

As a 用户,
I want 系统指出候选结果中的语义冲突和可复现性风险，并给出修复建议,
so that 我能理解结果为什么不可靠以及如何修正。

## Acceptance Criteria

1. QA Critic 检测内部语义冲突和可复现性不足，并以标准化 `issues` 与 `fixes` 返回。
2. 报告展示严重程度、影响字段、问题来源阶段与影响范围。
3. MVP 首版至少覆盖关键风格冲突、字段不一致和复现不足三类高频问题。

## Tasks / Subtasks

- [ ] 定义 semantic conflict 与 reproducibility issue 分类。
- [ ] 实现问题检测规则和修复建议生成器。
- [ ] 将问题与来源阶段、影响范围关联。
- [ ] 在 QA 报告中输出 verdict、issues、fixes。
- [ ] 为规则与报告渲染补齐测试。
