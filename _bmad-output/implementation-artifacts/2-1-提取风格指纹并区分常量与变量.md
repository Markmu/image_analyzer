# Story 2.1: 提取风格指纹并区分常量与变量

Status: ready-for-dev

## Story

As a 用户,
I want 系统从客观描述中提取风格指纹并区分风格常量与可替换变量,
so that 我能理解哪些风格特征应被锁定，哪些内容允许修改。

## Acceptance Criteria

1. 从 `objective_description` 输出 `style_fingerprint`，并明确区分 `locked_style_constants` 与 `user_variables`。
2. 至少覆盖 lighting、color palette、camera language 等关键风格维度。
3. 字段命名与结构满足统一 IR 契约，可直接供变量编辑器和 Prompt Compiler 使用。

## Tasks / Subtasks

- [ ] 设计 `style_fingerprint` schema，并拆分常量、变量和关键风格维度。
- [ ] 实现从 `objective_description` 到 `style_fingerprint` 的转换服务。
- [ ] 明确定义 `locked_style_constants` / `user_variables` 命名边界。
- [ ] 将结果持久化为阶段快照，暴露给后续 controls 与 prompt compiler。
- [ ] 为转换逻辑与 schema 增加单测。

## Dev Notes

### Technical Requirements

- 本 story 只负责风格指纹提取，不要引入 prompt adapter、QA verdict 或最终结果拼装。
- 变量和常量必须结构化分离，不能靠文案约定或注释区分。
- `provider` 与 `adapter_type` 是后续阶段概念，当前阶段不要提前混用这些词。
