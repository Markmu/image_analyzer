# Story 5.1: 保存模板并重载历史任务结果

Status: ready-for-dev

## Story

As a 用户,
I want 保存分析输出模板并从历史任务中重载结果,
so that 我可以复用已有成果，而不必每次从头开始分析。

## Acceptance Criteria

1. 用户可保存可信结果对应模板及必要元数据，并在后续会话再次读取。
2. 用户可从历史任务中重载规范化结果视图和模板上下文，不需要重新执行完整分析链路。
3. 历史与模板接口只消费规范化任务结果视图，不直接拼装阶段快照。

## Tasks / Subtasks

- [ ] 定义模板保存 payload 和历史重载 view model。
- [ ] 让模板服务从最终结果对象中提取可复用数据。
- [ ] 更新历史重载流程，恢复模板上下文与工作区状态。
- [ ] 补齐模板保存、历史读取、权限校验测试。

## Dev Notes

### Technical Requirements

- 模板层消费 `prompt_outputs` 和模板视图模型，不读取内部快照。
- 历史重载恢复的是规范化结果对象，而非旧 `analysisData`。
- 不复用 support/search 接口实现普通用户历史查询。

### Architecture Compliance

- 继续使用 `features/templates/*` 与 `features/history/*` 分层。
- 如需兼容旧历史表，提供转换层，不要让新主链路依赖旧结构。

### Library / Framework Requirements

- React Query 适合作为历史列表和详情查询层。
- Drizzle schema 变更应围绕新任务模型与模板元数据。
