# Story 3.2: 生成三类适配表达

Status: ready-for-dev

## Story

As a 用户,
I want 获得 Natural Language、Tag Stack 和 Short Command 三类适配表达,
so that 我可以根据外部生图模型或个人偏好选择合适的提示形式。

## Acceptance Criteria

1. 返回 Natural Language、Tag Stack、Short Command 三类输出，并都与同一份风格模板保持语义一致。
2. `adapter_type` 命名只表示输出形式，不能与 provider 混用。
3. 用户在同一任务/页面中切换查看三类表达，无需创建额外资源。

## Tasks / Subtasks

- [ ] 设计 `prompt_outputs.adapters` 结构与 `adapter_type` 枚举。
- [ ] 基于同一 universal template 实现三类 adapter 转换器。
- [ ] 在结果页中支持适配表达切换，不重新请求独立资源。
- [ ] 为 adapter 输出和切换 UI 补齐测试。

## Dev Notes

### Technical Requirements

- `adapter_type` 允许值至少包括 `natural_language`、`tag_stack`、`short_command`。
- 所有 adapter 结果共享同一份任务结果对象，不拆成多个表或多个任务。
- Adapter 层不直接接触数据库连接或请求对象。

### Architecture Compliance

- Adapter 逻辑建议落在 `src/lib/analysis-ir/adapters/`。
- 前端切换逻辑放在 `src/features/analysis` 或 `src/features/templates` 结果工作区组件中。
- 不要把现有 provider router 目录当作 prompt output adapter 目录使用。

### Library / Framework Requirements

- React Query 读取统一结果对象；本地状态只保存当前选中的 `adapter_type`。
- Zod 校验 adapters payload。

### File Structure Requirements

- Adapters: `src/lib/analysis-ir/adapters/{natural-language,tag-stack,short-command}.ts`
- Shared enums/types: `src/lib/analysis-ir/types.ts`
- UI: `src/features/analysis/components/PromptAdapterSwitcher.tsx`

### Testing Requirements

- 单测覆盖三类 adapter 的结构与关键语义一致性。
- UI 测试覆盖切换表达类型时不丢失任务上下文。

### Previous Story Intelligence
