# Story 1.3: 生成完整结构化客观描述结果

Status: ready-for-dev

## Story

As a 用户,
I want 在任务完成后获得完整的结构化客观图像描述,
so that 我能基于可见事实理解图片内容并为后续风格提取提供完整输入。

## Acceptance Criteria

1. 返回完整 `objective_description`，至少包含可见内容描述、摄影/成像特征、整体置信度和不确定字段列表。
2. 无法可靠判断的字段显式标记为 `unknown` 或等价结构，不允许静默省略。
3. `objective_description` 满足统一 JSON 契约，并保持 WCAG 2.1 AA 下的结果可读性。

## Tasks / Subtasks

- [ ] 为 `objective_description` 设计 Zod schema 与 TypeScript 推导类型。
- [ ] 在分析阶段实现 Forensic Describer，输出可见事实、摄影特征与 uncertainty list。
- [ ] 将阶段产物持久化为 stage snapshot，并在结果读取接口中返回本阶段结果。
- [ ] 更新结果 UI，使其明确区分确定字段与 `unknown` 字段。
- [ ] 为 schema、阶段服务、结果渲染补齐测试。

## Dev Notes

- 本 story 只交付 `objective_description`，不要提前把 `style_fingerprint` 或 `qa_report` 混入该阶段。
