# Story 6.2: 渲染可索引内容页并提供工具回流入口

Status: ready-for-dev

## Story

As a 搜索用户,
I want 查看结构清晰、可索引的公开分析结果页,
so that 我能快速理解某个风格案例并进入工具主流程继续使用产品。

## Acceptance Criteria

1. 页面展示标题、摘要、关键风格块和工具回流入口，且与工具内核心分析结果保持一致但不暴露内部字段。
2. 页面采用适合索引的渲染方式，输出稳定的标题层级、摘要块、关键风格块和可访问链接结构。
3. 用户无需登录即可查看公开页，但进入工具主流程时按现有平台规则处理。

## Tasks / Subtasks

- [ ] 基于 public result projector 输出实现内容页数据加载。
- [ ] 渲染稳定的 SEO 结构、标题层级和 CTA。
- [ ] 确保公开内容与工具页共享同一核心结果语义。
- [ ] 为索引友好渲染和公开访问补齐测试。

## Dev Notes

### Technical Requirements

- 页面只能消费公开投影，不得访问内部回放对象或任务明细 JSONB。
- 公开页和工具页共享同一分析结果语义，但呈现层可以不同。
- CTA 需明确回流到主分析工具入口。

### Architecture Compliance

- 页面建议放在 `src/app/library/[id]/page.tsx` 或新的公开内容路由。
- Metadata/SEO 从 public result 生成，不从内部任务对象生成。

### Library / Framework Requirements

- Next.js App Router 适合 SSR/静态化公开内容页。
- 页面元数据使用官方 metadata API。

### File Structure Requirements

- Public page: `src/app/library/[id]/page.tsx`
- Presenter consumption helpers: `src/features/library/lib/public-result-loader.ts`
- Components: `src/features/library/components/*` 或延续现有 library 页面组件

### Testing Requirements

- 页面测试覆盖公开访问、SEO 结构、CTA 跳转、禁止内部字段渲染。
- 可加入简单快照测试确保标题层级稳定。
