# Story 6.1: 生成公开结果投影并执行字段白名单控制

Status: ready-for-dev

## Story

As a 系统,
I want 将已批准公开的分析结果转换为公共结果投影,
so that 内容站页面可以安全复用分析结果而不暴露内部调试信息。

## Acceptance Criteria

1. 公开结果对象由专用 `public-result-projector` 生成，只包含白名单字段。
2. 内部字段如 `input_payload`、`analysis_stage_snapshots`、完整 `qa_report.issues/fixes`、支持排障字段和审计字段不得公开。
3. 公开投影采用 allowlist 构建，而不是删字段式过滤。

## Tasks / Subtasks

- [ ] 定义 public result schema 和 allowlist。
- [ ] 实现 `public-result-projector`，从最终结果对象生成公开投影。
- [ ] 校验公开标记和发布前置条件。
- [ ] 为禁止公开字段添加防回归测试。

## Dev Notes

### Technical Requirements

- 公开投影必须从稳定的最终结果对象生成，不能直接读取内部 snapshots。
- allowlist 是唯一允许暴露字段来源，禁止 subtractive filtering。
- 只暴露标题、摘要、关键风格块、CTA 等必要字段。

### Architecture Compliance

- projector 推荐 `src/lib/presenters/public-result-projector.ts`。
- 公开结果可落到独立 read model 或按需生成，但边界必须固定。

### Library / Framework Requirements

- Zod 4 校验公开结果 shape。

### File Structure Requirements

- Tests: `src/lib/presenters/__tests__/public-result-projector.test.ts`
