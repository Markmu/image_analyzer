# Story 2.2: 提供标准变量槽位并支持安全替换

Status: ready-for-dev

## Story

As a 用户,
I want 使用标准变量槽位替换主体、场景、时间和画幅,
so that 我可以复用模板而不破坏参考图的核心风格。

## Acceptance Criteria

1. 提供主体、场景、时间、画幅四类标准变量槽位，并只允许编辑白名单字段。
2. 用户不能直接修改 `locked_style_constants`，且界面说明不可编辑原因。
3. 替换后的结构仍可被后续 prompt 编译阶段直接消费。

## Tasks / Subtasks

- [ ] 定义 controls/schema，明确变量白名单与锁定字段。
- [ ] 为变量编辑器设计输入校验与错误反馈。
- [ ] 在 workspace 层保存用户临时替换状态，但任务真相仍走服务端结果。
- [ ] 增加服务端校验，拒绝对锁定字段的非法修改。
- [ ] 为 controls 结果和 UI 交互补齐测试。

## Dev Notes

### Technical Requirements

- 变量替换属于 controls 阶段，不要把 editor 本地状态误认为持久化真相。
- 所有可替换字段必须由 allowlist 驱动，拒绝自由路径写入。
- 保持输出与后续 `prompt_outputs` 编译兼容，不引入临时 UI 专属字段。

### Architecture Compliance

- 推荐将服务端 controls 逻辑放在 `src/lib/analysis-ir/stages/controls.ts`。
- 前端局部状态可扩展 `src/features/templates/stores/useTemplateEditorStore.ts` 或新增 analysis workspace store，但不得存任务真相。
- Route/service 层要与 `src/lib/api/response.ts` 的统一响应约定保持一致。

### Library / Framework Requirements

- React Hook Form + Zod 适合变量表单校验。
- Zustand 只存草稿与界面选择，最终提交值通过 API 或 server action 校验。

### File Structure Requirements

- Stage: `src/lib/analysis-ir/stages/controls.ts`
- Schemas: `src/lib/analysis-ir/schemas/controls.ts`
- UI/editor: `src/features/analysis/components/VariableEditor.tsx`
- Store: `src/features/analysis/stores/useAnalysisWorkspaceStore.ts` 或等价路径

### Testing Requirements

- 校验白名单变量可编辑、锁定字段被拒绝、非法路径被拒绝。
- 交互测试覆盖键盘输入、错误提示、提交后工作区状态保持。
