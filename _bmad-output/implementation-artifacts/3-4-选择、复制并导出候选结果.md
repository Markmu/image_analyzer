# Story 3.4: 选择、复制并导出候选结果

Status: ready-for-dev

## Story

As a 用户,
I want 选择适配表达类型与强度档位，并复制或导出候选结果,
so that 我可以把这些输出直接用于外部生图工作流验证。

## Acceptance Criteria

1. 用户可切换表达类型和强度档位，并保持当前工作区选择状态。
2. 系统至少支持复制当前结果文本，并导出包含所选 `adapter_type`、强度档位和参数建议的结构化内容。
3. 复制/导出结果不包含敏感凭据或内部调试字段，操作后界面给出成功反馈且不丢上下文。

## Tasks / Subtasks

- [ ] 设计导出 payload，仅包含允许公开给用户的 prompt 输出与参数建议。
- [ ] 实现复制与导出入口，并绑定当前 adapter/variant 选择。
- [ ] 过滤内部调试字段和敏感信息。
- [ ] 为成功/失败反馈、复制、下载补齐测试。

## Dev Notes

### Technical Requirements

- 导出只暴露规范化 `prompt_outputs` 视图，不读取内部阶段快照。
- 不能把 R2 凭据、内部错误、原始调试信息写入导出文件或复制内容。
- 当前选择状态应留在工作区，不通过重建任务来切换。

### Architecture Compliance

- 导出逻辑可沿用 `src/features/templates/lib/template-exporter.ts` 的结构，但需适配新 IR。
- 结果页组件只消费最终结果对象，不直连 stage snapshots。

### Library / Framework Requirements

- 浏览器复制可用 `navigator.clipboard`，失败时提供降级反馈。
- 导出内容可优先使用 JSON 或文本文件，结构需稳定。

### File Structure Requirements

- UI/components: `src/features/analysis/components/PromptExportPanel.tsx`
- Export helpers: `src/features/templates/lib/prompt-output-exporter.ts`
- Hooks: `src/features/templates/hooks/useExportPromptOutputs.ts`

### Testing Requirements

- 单测覆盖导出过滤逻辑。
- 组件测试覆盖复制成功/失败反馈、导出 payload 结构、选择状态保持。
