# Story 1.1: 单图上传与任务初始化

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 内容创作者,
I want 上传一张参考图并立即创建分析任务,
so that 我可以快速进入风格分析流程。

## Acceptance Criteria

1. Given 用户具备任务创建权限并处于可上传页面，When 用户上传合法图片并提交，Then 系统创建唯一任务 ID 并返回初始任务状态，And 任务与用户、图片资源建立可追踪关联。
2. Given 用户上传图片格式或大小不合法，When 系统执行输入校验，Then 系统拒绝任务创建并返回可操作错误信息，And 页面提供重新上传入口。

## Dev Notes

Story 1.1 是 Epic 1 的入口故事，目标是把“上传参考图 -> 建立任务链路”做成稳定起点，避免后续分析链路无任务可追踪。

### Developer Context

- 本故事只负责“单图上传 + 任务初始化”，不实现客观描述引擎本体（属于 Story 1.2）。
- 任务状态初始化必须可被前端轮询读取（Story 1.3 将消费）。
- 必须保证任务、用户、图片资源三者关联可追溯，为后续回放与支持排障打基础。

### Technical Requirements

- API 响应必须统一使用 `{ success, data|error }` 契约。
- 上传失败与校验失败错误必须可操作，至少包含清晰 message 和可执行下一步。
- 任务初始化时生成唯一任务标识（UUID/等价唯一 ID），并设置初始状态。
- 上传入口必须覆盖桌面（拖拽+点击）与移动（拍照/相册）。
- 状态变化需要支持轮询读取，避免前端进入黑盒等待。

### Architecture Compliance

- 保持 Next.js App Router + API Routes 模式，不新增并行架构。
- 数据层遵循 PostgreSQL + Drizzle 规范，命名使用 `snake_case`。
- 前端状态策略保持既有约束：UI 状态可用 Zustand，服务器状态使用 React Query。
- 上传与任务初始化流程必须按现有边界组织：`features/*` + `lib/*`。

### Library / Framework Requirements

- Next.js：`16.1.6`（npm registry 当前稳定版本）
- next-auth：`4.24.13`
- @tanstack/react-query：`5.90.21`
- drizzle-orm：`0.45.1`
- zustand：`5.0.11`

说明：如仓库当前锁定版本不同，以仓库已安装版本优先，不在本故事中发起大版本升级。
