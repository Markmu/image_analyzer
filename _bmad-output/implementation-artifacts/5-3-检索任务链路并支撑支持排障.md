# Story 5.3: 检索任务链路并支撑支持排障

Status: ready-for-dev

## Story

As a 支持人员,
I want 按任务 ID、时间范围和异常类型检索任务链路,
so that 我能在 5 分钟内定位问题发生在分析、编译还是 QA 阶段。

## Acceptance Criteria

1. 支持角色可按任务 ID、时间范围和异常类型搜索，单次最多返回 100 条结果。
2. 搜索结果返回阶段输出、版本信息和 QA 问题列表，并遵守授权与审计边界。
3. 搜索结果默认摘要优先，并保留进入详细阶段快照的受控入口。

## Tasks / Subtasks

- [ ] 设计 support search API 与查询参数 schema。
- [ ] 实现 summary-first 搜索结果和详情跳转能力。
- [ ] 增加支持角色权限和审计日志记录。
- [ ] 为 5 分钟定位目标设计高信号摘要字段。
- [ ] 补齐搜索、RBAC、审计测试。

## Dev Notes

### Technical Requirements

- `support/search` 与普通用户接口隔离，不默认返回全量 payload。
- 审计日志至少包含 `actor_id`、`task_id`、`action`、`timestamp`、`reason`。
- 支持角色默认只读，不允许修改任务和结果。

### Architecture Compliance

- API 推荐 `src/app/api/analysis/support/search/route.ts`。
- service 推荐 `src/lib/analysis-tasks/support-search-service.ts`。
- 搜索结果应依赖回放与最终结果 read model，不绕过它们直接拼散乱查询。

### Library / Framework Requirements

- Zod 校验查询参数与响应摘要结构。
- 鉴权延续 Auth.js 宿主能力。

### File Structure Requirements

- API: `src/app/api/analysis/support/search/route.ts`
- Service: `src/lib/analysis-tasks/support-search-service.ts`
- Audit helpers: `src/lib/audit/support-access.ts`

### Testing Requirements

- 覆盖未授权拒绝、摘要结果、100 条上限、reason 必填。
- 集成测试覆盖问题定位字段是否足够定位阶段。
