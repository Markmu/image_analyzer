# Executive Summary

> **项目：** image_analyzer UX 设计规范  
> **版本：** v2.0  
> **最后更新：** 2026-03-01

---

## Project Vision

image_analyzer 的核心体验已经从“图片分析工具”收敛为“图片到风格提示词的结构化编译器”。

用户上传一张参考图，不是为了得到一段泛化描述，而是为了得到一组可复用、可替换、可审计的结果资产：

- 客观图像勘验
- 风格指纹
- 风格常量与变量槽位
- 多适配器 Prompt 输出
- QA 校验与修复建议

产品价值不在“描述图片”，而在“让用户替换主体或场景后，依然能稳定复现原图风格”。

**一句话价值主张：**  
帮助用户把参考图转成可以跨模型迁移、可重复使用的风格模板，而不是一次性的提示词文本。

---

## Target Users

### 1. 创作者与设计师

- 需要快速复现参考图风格
- 更关注输出是否可直接投入外部生图工作流
- 需要变量替换、强度切换、模板保存与历史重载

### 2. 新手用户

- 不懂摄影、配色或提示词工程术语
- 需要系统帮他把“复杂的风格结构”翻译成可理解、可使用的操作结果
- 更依赖默认推荐和清晰的主操作

### 3. 支持与运营人员

- 需要按任务回放链路定位问题
- 需要区分“分析问题、编译问题、模型适配问题、QA 问题”
- 需要安全地消费公开结果页和任务摘要

---

## Product Scope That UX Must Respect

### MVP 内

- 单图上传与任务创建
- 异步任务进度展示
- 结构化结果对象展示
- 变量替换与风格常量提示
- Adapter 与强度切换
- 模板复制、导出、保存、历史重载
- QA verdict、issues、fixes 的用户可读展示
- 最小公开结果页投影

### 明确不在 MVP

- 多图联合分析
- 内置生图结果作为核心验收项
- 团队协作与模板市场
- 重新设计登录、计费、Credit 体系

这意味着 UX 不能再把多图分析、社交证明、社区展示或“成功率 94%”类信号当成主叙事。

---

## Key UX Challenges

### 1. 让结构化链路可理解，但不把用户拖进技术细节

架构里有 `objective_description`、`style_fingerprint`、`controls`、`prompt_outputs`、`qa_report`。  
用户不需要背下这些名字，但需要理解：

- 系统先看见什么
- 再提炼什么
- 最后给他什么可用结果

### 2. 让“可复现”变成可感知

用户必须明确知道：

- 哪些风格特征会被锁定
- 哪些内容允许替换
- 换主体不等于换风格

### 3. 在等待阶段建立信任，而不是制造神秘感

旧版“专业术语滚动 + 社会证明”已经不够。  
新版更需要：

- 阶段进度
- 当前步骤说明
- 用户可见的任务状态
- 错误与重试的可解释反馈

### 4. 结果页既要服务用户，也要服务后续系统

同一个任务结果要同时支持：

- 用户复制和导出
- 支持人员回放
- 历史重载
- 内容站公开投影

UX 必须用清晰的信息分层来避免把内部对象直接“倒给”用户。

---

## UX Opportunities

### 1. 从“结果页”升级为“风格工作台”

用户看到的不应只是静态分析报告，而应是一个可以：

- 查看结构化结果
- 切换 Adapter / intensity
- 替换变量
- 复制/导出
- 回放历史

的工作台。

### 2. 把 QA 从“内部门禁”变成“用户信心增强器”

QA verdict 不只是工程字段，它可以帮助用户判断：

- 当前模板是否可以直接使用
- 是否存在风格冲突
- 是否建议先调整变量或强度

### 3. 把公开结果页变成产品入口，而不是调试结果泄露面

公开结果页只展示 allowlist 投影后的摘要、风格块和 CTA。  
这给 UX 一个明确边界：公共页面是“解释价值”，不是“暴露原始链路”。

---

## Success Metrics UX Translation

PRD 指标需要被翻译成可设计、可观测的体验目标：

- 采纳率 >= 65%
  - 首屏主操作必须清晰，结果结构必须易于复制/导出
- 生成满意率 >= 60%
  - 变量替换和风格常量提示必须足够清楚
- 模板二次编辑率 <= 35%
  - 默认输出应尽量可直接用
- 端到端 P95 <= 45 秒
  - 等待态必须透明，不能让用户误以为卡死

---

## UX Direction Summary

新版 UX 的核心，不再是“炫技式展示分析能力”，而是：

- 用任务进度建立信任
- 用结构化结果降低编辑成本
- 用变量槽位保护风格迁移成功率
- 用 QA 与回放机制保障可解释性
- 用统一设计 token 保持工具页与公开页视觉一致
