# UX Consistency Patterns

> **项目：** image_analyzer UX 设计规范  
> **版本：** v2.0  
> **最后更新：** 2026-03-01

---

## 目标

一致性规范的目标，不是让所有页面“长得一样”，而是让用户在任何任务阶段都能快速理解：

- 当前状态是什么
- 主要动作是什么
- 风险在哪里
- 哪些区域可展开，哪些内容只是说明

---

## Button Hierarchy

### 一级按钮

用于当前上下文里的唯一主操作：

- 上传并开始任务
- 复制当前 Prompt
- 导出结果

**视觉原则：**

- 使用 `--primary`
- 深色背景上保持高对比
- 允许轻微蓝色光晕
- 文案优先，图标辅助

### 二级按钮

用于重要但非首选动作：

- 保存模板
- 查看修复建议
- 重新分析

**视觉原则：**

- 玻璃态背景 + 主色边框
- 不抢主按钮权重

### 三级按钮

用于低风险辅助动作：

- 取消
- 返回
- 查看详情
- 展开更多

**视觉原则：**

- 文本按钮或低强调按钮
- 保持可见，不制造视觉噪音

### 危险按钮

用于删除、放弃、覆盖等高风险动作：

- 使用 `--error`
- 必须配确认或明显的风险上下文

---

## Feedback Patterns

### Toast

用于短反馈：

- 复制成功
- 导出成功
- 保存成功
- 上传失败

### Inline Feedback

用于与当前内容强绑定的解释：

- QA `warn` / `fail`
- 变量输入错误
- 结果暂不可导出

### Persistent Status

用于任务级状态：

- queued
- running
- completed
- failed
- canceled

任务状态不能只依赖 Toast，必须在页面固定位置持续可见。

---

## Status Semantics

| 状态 | 颜色 | 语义 |
|------|------|------|
| `pass` / `completed` | `--success` | 结果可直接使用或流程完成 |
| `running` | `--primary` + `--accent` | 任务进行中 |
| `warn` | `--warning` | 结果可用但建议复核 |
| `fail` / `error` | `--error` | 结果不可直接使用或任务失败 |
| `disabled` | `--secondary` | 当前不可操作 |

规则：

- 状态必须同时包含颜色、图标、文字
- `warn` 不能伪装成成功态
- QA 状态与任务状态分开表达

## Form Patterns

### 上传区

- 默认显示拖拽与点击双入口
- hover / dragover 使用蓝色高亮
- 上传失败后保留上下文，不清空用户选择

### 变量输入

- 风格常量与变量槽位必须视觉区分
- 变量占位使用 `{subject}` 这类结构，而不是示例性自然语言占位
- 输入框下方可展示边界说明，如“替换主体，不建议改动光型”

### 选择器

适用于：

- Adapter 切换
- intensity 切换
- 导出格式切换

要求：

- 当前选中态清晰
- 不能只靠颜色表达选中
- 切换后结果变化要立即反映在输出区

---

## Disclosure Patterns

### 默认展开

- 当前 Prompt
- QA verdict
- 当前任务状态

### 默认折叠

- QA issues 详情
- fixes 详情
- 阶段回放
- 公开结果投影预览

### 交互规则

- 折叠标题必须简洁直接
- 展开后不能导致页面主操作完全移出视口
- 展开/收起应使用 200-300ms 动效

---

## Navigation Patterns

### 顶层导航

顶层导航应面向产品范围，而不是旧版“分析 / 模版库 / 账户”固定结构。  
推荐按真实功能域组织：

- 分析任务
- 历史记录
- 模板
- 公开结果
- 设置

### 页面内导航

主工作台优先使用：

- 面板结构
- Tab
- Accordion

而不是大量跳页。

---

## Empty, Loading, Error States

### Empty

- 明确告诉用户下一步能做什么
- 不使用过度装饰性文案

### Loading

- 显示阶段进度，而不是只有 Spinner
- 长耗时时要有阶段说明

### Error

- 说明失败发生在哪个层级
- 提供可恢复动作：重试、返回、查看说明

---

## Consistency Checklist

- [ ] 主按钮始终只服务一个最重要动作
- [ ] QA 状态和任务状态分开表达
- [ ] 变量槽位与锁定常量不会混淆
- [ ] 选中态不只依赖颜色
- [ ] 长耗时状态有阶段说明
- [ ] 错误提示都带恢复动作
