# 核心流程优化方案

> **项目：** image_analyzer UX 设计规范  
> **版本：** v2.0  
> **最后更新：** 2026-03-01

---

## 目标

把原来的“上传 -> 等待 -> 看结果”单线流程，升级为**围绕异步分析任务展开的可解释工作流**。

新版流程必须同时满足：

- 用户知道系统做到哪一步了
- 用户知道当前结果是否可以直接使用
- 用户知道哪些内容能改、哪些不该改
- 用户能顺畅完成复制、导出、保存和重试

---

## 新版核心流程

### 4 步主链路

```text
1. 上传并创建任务
2. 查看阶段进度
3. 消费结构化结果
4. 采取行动（复制 / 导出 / 保存 / 重试）
```

---

## Step 1: 上传并创建任务

### 用户动作

- 拖拽图片到上传区域
- 或点击选择图片

### 系统响应

- 立即显示上传进度
- 上传完成后立即创建 `analysis_task`
- 页面切换到任务上下文，不要求用户二次确认

### 视觉反馈

- 拖拽时使用蓝色高亮边框，不使用绿色虚线
- 上传中显示进度和文件信息
- 任务创建成功后展示任务头部

### 错误处理

- 上传失败：留在上传态，显示重试入口
- 任务创建失败：保留已上传信息，允许重新提交

---

## Step 2: 查看阶段进度

### 用户应该看到什么

- 当前阶段
- 总体进度
- 当前说明文案
- 是否支持取消或重试

### 阶段顺序

| 阶段 | 用户可见文案 | 目的 |
|------|-------------|------|
| `forensic_describer` | 图像勘验中 | 提取可见事实 |
| `style_fingerprinter` | 提炼风格指纹中 | 识别稳定风格结构 |
| `prompt_compiler` | 编译提示词模板中 | 生成可用输出 |
| `qa_critic` | 执行质量校验中 | 检查冲突与修复建议 |

### 进度态要求

- 每个阶段变化都要伴随文案变化
- 不使用“神秘 AI 术语跑马灯”作为主要反馈
- 允许在长耗时场景中展示“正在优化输出稳定性”

---

## Step 3: 消费结构化结果

结果页信息层级必须重排为：

### 首层：立刻可用

- 当前推荐 Prompt
- Adapter / intensity 切换
- 复制
- 导出
- 保存模板
- QA verdict

### 第二层：帮助理解

- 风格概览
- 锁定风格常量
- 变量槽位
- 参数建议

### 第三层：帮助排障和回放

- 结构化分析块
- QA issues / fixes 细节
- 阶段快照入口
- 公开结果页投影预览

---

## Step 4: 采取行动

### 主操作

- 复制当前 Prompt
- 导出结果
- 保存模板

### 次操作

- 重新分析
- 切换强度
- 切换表达形式
- 查看 QA 修复建议

### 高级操作

- 历史重载
- 回放阶段结果
- 查看公开结果页投影

---

## 推荐首屏结构

```text
任务头部
-> 阶段进度
-> 当前推荐 Prompt
-> Adapter / intensity 切换
-> QA verdict
-> 风格概览
```

这样用户无需滚动太多，就能完成“判断是否能用 + 直接带走结果”。

---

## 旧流程中的问题与新版修正

| 旧问题 | 新版修正 |
|-------|---------|
| 上传完后等待感强 | 创建任务后立即进入任务上下文 |
| 结果先堆分析细节 | 先展示可用模板和主操作 |
| 强依赖四维度卡片 | 改为风格概览 + 结构化块分层 |
| 用社会证明建立信任 | 改为阶段透明度 + QA verdict + 可回放性 |
| 移动端与桌面端割裂 | 改为统一响应式工作台 |

---

## 移动端裁剪策略

移动端不是“桌面端的阉割版”，而是“同一任务事实的压缩视图”。

### 移动端默认保留

- 任务状态
- 当前阶段
- 当前推荐 Prompt
- 复制按钮
- QA verdict
- 风格概览

### 移动端默认折叠

- 锁定常量详情
- 变量编辑器扩展说明
- 结构化分析块
- 回放入口
- 公开结果预览

---

## 验收标准

### 体验层

- 用户在上传后 1 秒内看到任务状态变化
- 用户在结果首屏能直接看到“复制 / 导出 / 保存”
- 用户能在首屏判断结果是 `pass / warn / fail`
- 用户能明确识别变量槽位

### 结构层

- 页面显示顺序与 IR 主链路一致
- 公开结果预览不暴露内部调试字段
- QA 结果不依赖用户自行推断

---

## 开发检查清单

- [ ] 上传完成后立即创建任务并切换到任务视图
- [ ] 阶段文案与真实后端阶段一致
- [ ] 首屏包含复制、导出、保存
- [ ] QA verdict 首屏可见
- [ ] 变量槽位视觉上与锁定常量明确区分
- [ ] 移动端不隐藏任务真相，只折叠深层信息
